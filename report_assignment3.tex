\begin{abstract}
This report presents an empirical study of the relationship between regularization strength,
algorithmic stability, and generalization performance in linear regression.
Following the theoretical framework of Bousquet and Elisseeff~\cite{bousquet2002},
we implement Ridge regression, ordinary least squares (OLS), and Lasso from scratch
and measure uniform stability via a leave-one-out prediction-change metric.
Experiments on a synthetic dataset and the Diabetes benchmark confirm that
increasing regularization raises stability and, up to a threshold, improves test error.
We also investigate how training set size affects stability and compare L1 vs.\ L2
regularization behavior.
\end{abstract}

\section{Introduction}
% ============================================================

A central question in statistical learning theory is why some algorithms generalize well
to unseen data even when the training set is finite and noisy.
The classical answer, rooted in Vapnik-Chervonenkis (VC) theory, bounds generalization
error using the complexity of the hypothesis class.
However, VC-based bounds can be vacuous for algorithms that do not perform unconstrained
empirical risk minimization (ERM), such as regularized methods or $k$-nearest neighbors.

Bousquet and Elisseeff~\cite{bousquet2002} propose an orthogonal perspective:
\emph{algorithmic stability}.
Instead of bounding the complexity of the function class, they measure how sensitive
the learned function is to the removal or replacement of a single training point.
They prove that a stable algorithm has a small generalization gap, and crucially, that
regularization-based algorithms such as Ridge regression and SVMs are stable with
stability $\beta = \mathcal{O}(1/\lambda m)$, where $\lambda$ is the regularization
parameter and $m$ is the training set size.

This assignment connects these theoretical insights to practice by:
\begin{itemize}
    \item implementing Ridge regression, OLS, and Lasso from scratch;
    \item estimating stability empirically via a leave-one-out prediction-change metric;
    \item studying how stability and generalization co-vary as $\lambda$ changes;
    \item comparing L1 vs.\ L2 regularization and analyzing the effect of dataset size.
\end{itemize}

\section{Theoretical Background}
% ============================================================

\subsection{Setup and Notation}

Let $S = \{(x_1, y_1), \ldots, (x_m, y_m)\} \subset \mathcal{X} \times \mathcal{Y}$
be a training set drawn i.i.d.\ from an unknown distribution $\mathcal{D}$.
A learning algorithm $A$ maps $S$ to a function $\AS : \mathcal{X} \to \mathcal{Y}$.
Given a loss function $\ell(f, z) = c(f(x), y)$, the quantities of interest are:

\begin{align}
    R(A, S)      &= \E_z[\ell(\AS, z)]             && \text{(generalization error)} \\
    R_\text{emp} &= \frac{1}{m}\sum_{i=1}^m \ell(\AS, z_i) && \text{(training error)} \\
    R_\text{loo} &= \frac{1}{m}\sum_{i=1}^m \ell(\ASi, z_i) && \text{(leave-one-out error)}
\end{align}

where $S^{\setminus i}$ denotes the training set with the $i$-th point removed.

\subsection{Notions of Stability}

Bousquet and Elisseeff introduce several notions of stability, the most important being:

\begin{definition}[Uniform Stability~\cite{bousquet2002}]
An algorithm $A$ has \emph{uniform stability} $\beta$ with respect to a loss $\ell$ if
\[
    \forall S \in \mathcal{Z}^m,\; \forall i \in \{1,\ldots,m\},\quad
    \sup_{z \in \mathcal{Z}} \abs{\ell(\AS, z) - \ell(\ASi, z)} \leq \beta.
\]
\end{definition}

Uniform stability implies the weaker notions of hypothesis stability and error stability.
An algorithm is called \emph{stable} when $\beta_m = \mathcal{O}(1/m)$.

\subsection{Generalization Bound for Stable Algorithms}

The main result of~\cite{bousquet2002} connects uniform stability to generalization:

\begin{theorem}[Theorem 12 in~\cite{bousquet2002}]
Let $A$ be an algorithm with uniform stability $\beta$ and bounded loss
$0 \leq \ell(\AS, z) \leq M$.
Then, for any $m \geq 1$ and $\delta \in (0,1)$, with probability at least $1-\delta$:
\[
    R \;\leq\; R_\text{emp} + 2\beta + (4m\beta + M)\sqrt{\frac{\ln(1/\delta)}{2m}}.
\]
\end{theorem}

The bound is non-trivial when $\beta = \mathcal{O}(1/m)$: the first term
$2\beta \to 0$ while the second term decreases as $\mathcal{O}(1/\sqrt{m})$.

\subsection{Stability of Ridge Regression}

For the algorithm
\[
    \AS = \arg\min_{w \in \R^d} \frac{1}{m}\sum_{i=1}^m (w^\top x_i - y_i)^2
          + \lambda \norm{w}^2_2,
\]
Bousquet and Elisseeff (Example 3 in~\cite{bousquet2002}) show that the uniform
stability satisfies
\[
    \beta \;\leq\; \frac{2\kappa^2 B^2}{\lambda m},
\]
where $\kappa^2 = \sup_x k(x,x)$ is the kernel's maximum and $B$ bounds the labels.
This confirms that \emph{stronger regularization (larger $\lambda$) implies higher
stability (smaller $\beta$)}, linking the two concepts directly.

\section{Methods}
% ============================================================

\subsection{Algorithms Implemented from Scratch}

All algorithms are implemented using only \texttt{numpy}; no \texttt{sklearn}
estimator \texttt{.fit()} methods are used.

\paragraph{Ridge Regression (L2).}
Closed-form solution exploiting the matrix identity:
\[
    w_\lambda = (X^\top X + \lambda I)^{-1} X^\top y.
\]
Solved via \texttt{numpy.linalg.solve} for numerical stability.

\paragraph{Ordinary Least Squares (OLS).}
Treated as the limit $\lambda \to 0$, computed via the Moore-Penrose pseudoinverse
(\texttt{numpy.linalg.lstsq}).
Used as an unregularized baseline.

\paragraph{Lasso (L1).}
No closed form exists; we implement \textbf{coordinate descent} with
soft-thresholding. The update for coordinate $j$ is:
\[
    w_j \;\leftarrow\; \mathcal{S}_\lambda\!\left(\frac{X_{:,j}^\top r_j}{n}\right)
    \bigg/ \frac{\norm{X_{:,j}}^2}{n},
    \qquad
    \mathcal{S}_\lambda(t) = \operatorname{sign}(t)\max(\abs{t}-\lambda, 0),
\]
where $r_j = y - X w + X_{:,j} w_j$ is the partial residual.

\subsection{Stability Estimation}

We estimate uniform stability empirically as:
\begin{equation}
    \betahat \;=\; \frac{1}{m} \sum_{i=1}^{m}
    \frac{1}{|S_\text{test}|} \sum_{z \in S_\text{test}}
    \abs{f_S(x) - f_{S^{\setminus i}}(x)},
    \label{eq:stability_metric}
\end{equation}
where for each $i$ we remove the $i$-th training point, retrain the model, and measure
the average absolute change in predictions over the fixed test set $S_\text{test}$.
This directly approximates the hypothesis-stability definition in~\cite{bousquet2002}.

\subsection{Datasets}

\paragraph{Synthetic dataset.}
We generate $n = 300$ samples in $d = 20$ dimensions:
\[
    w^* \sim \mathcal{N}(0, I_d),\quad
    x_i \sim \mathcal{N}(0, I_d),\quad
    y_i = x_i^\top w^* + \varepsilon_i,\quad
    \varepsilon_i \sim \mathcal{N}(0, 1).
\]
This is a clean, controlled setting where the ground truth is known.
A 75/25 train-test split is used.

\paragraph{Diabetes dataset (real-world).}
The UCI Diabetes dataset (442 samples, 10 features) from \texttt{sklearn.datasets}
is used as the real-world benchmark.
Features are standardized to zero mean and unit variance; the target is also
standardized. A 75/25 train-test split is applied with no data leakage.

\paragraph{Motivation for dataset choice.}
The synthetic dataset allows controlled study of stability in a regime where the
data-generating process is known. The Diabetes dataset introduces real-world
structure, correlations between features, and non-idealities absent from synthetic
data, providing a more realistic test of the theory.

        and the bottom portion shows the dataset-size study.
    }
    \label{fig:main}
\end{figure}

\paragraph{Key observations.}

\begin{enumerate}
    \item \textbf{Stability increases with $\lambda$.}
    On both datasets, $\betahat$ decreases monotonically as $\lambda$ grows,
    consistent with the theoretical bound $\beta \leq 2\kappa^2 B^2 / (\lambda m)$.
    OLS (effectively $\lambda = 0$) shows the highest instability across all
    $\lambda$ values tested.

    \item \textbf{Test error follows a U-shaped curve.}
    For small $\lambda$, the model overfits and test error is high.
    As $\lambda$ increases, test error decreases (regularization reduces variance)
    until an optimal $\lambda^*$, after which bias dominates and test error rises again.
    This is the classical bias-variance tradeoff.

    \item \textbf{More stable models generalize better (up to a point).}
    Comparing stability and test error curves: the range of $\lambda$ where stability
    is high (large $\lambda$) largely overlaps with the range of good test performance,
    confirming the Bousquet-Elisseeff prediction.
    However, extreme regularization ($\lambda \gg \lambda^*$) increases stability
    further while \emph{hurting} test error due to underfitting — stability alone is
    not sufficient for good generalization; the algorithm must also fit the data.
\end{enumerate}

\subsection{Effect of Dataset Size on Stability}

\Cref{fig:main} (bottom-left panel) shows $\betahat$ vs.\ $\lambda$ for Ridge regression
trained on $n \in \{50, 150, 300\}$ samples from the synthetic dataset.

\paragraph{Observation.}
For a fixed $\lambda$, stability improves (smaller $\betahat$) as $n$ grows.
This is again consistent with theory: $\beta \propto 1/(\lambda m)$ implies that
both increasing $\lambda$ and increasing $m$ improve stability.
The curves for all three sizes converge at large $\lambda$ (regularization dominates)
and diverge at small $\lambda$ (sample size becomes the bottleneck).

\subsection{L1 vs.\ L2 Regularization}

\Cref{fig:main} (bottom-right panel) compares the stability of Ridge (L2) and
Lasso (L1) on the synthetic dataset.

\paragraph{Observation.}
Both methods become more stable as $\lambda$ increases, and their stability curves
are qualitatively similar. However, Lasso tends to be \emph{slightly less stable}
than Ridge for the same $\lambda$ value.
This can be explained theoretically: the L2 penalty induces a quadratic regularizer
whose Bregman divergence is the squared norm, leading to a tighter contraction bound
(Theorem 22 in~\cite{bousquet2002}).
The L1 penalty's non-differentiability means coordinate-descent solutions can change
more discontinuously when a single point is removed, resulting in higher instability.

\section{Discussion}
% ============================================================

\subsection{Linking Observations to the Theory}

The experiments directly validate the core message of~\cite{bousquet2002}:

\begin{itemize}
    \item \textbf{Regularization $\Rightarrow$ Stability.}
    The empirical stability metric $\betahat$ decreases with $\lambda$, mirroring the
    theoretical bound $\beta \leq 2\kappa^2 B^2 / (\lambda m)$.
    Unregularized OLS (whose pseudoinverse solution can change dramatically when
    one point is removed, especially near singularity) is consistently the least stable.

    \item \textbf{Stability $\Rightarrow$ Generalization (with caveats).}
    Stability is a \emph{sufficient} condition for generalization (Theorem 12 of~\cite{bousquet2002}),
    but not a necessary one.
    Our results show that the regime of good test performance (optimal $\lambda^*$)
    corresponds to an intermediate level of stability: neither too small
    (overfitting, high variance) nor so large that bias dominates.

    \item \textbf{Why OLS can still generalize.}
    Despite low stability, OLS achieves reasonable test error on both datasets at
    moderate problem dimensions.
    This is because stability bounds are sufficient but not tight; OLS can generalize
    well in benign settings even if its leave-one-out sensitivity is high.
\end{itemize}

\subsection{Limitations and Extensions}

\begin{itemize}
    \item \textbf{Computational cost.}
    Estimating $\betahat$ requires $m$ retraining steps per $\lambda$ value.
    For large datasets or high-dimensional problems, this is expensive.
    Approximate stability estimators (random subsampling) could reduce cost.

    \item \textbf{Stability metric choice.}
    We measure average absolute change in predictions (\cref{eq:stability_metric}).
    Alternative metrics (change in loss, worst-case change) may give different
    quantitative results, though qualitative trends should be similar.

    \item \textbf{Double descent.}
    In the overparameterized regime ($d > m$), OLS exhibits double descent
    (generalization improves after the interpolation threshold).
    Our experiments stay in the underparameterized regime; extending to $d \gg m$
    would reveal regimes where the classical stability analysis breaks down.
\end{itemize}

% ============================================================
\section{Conclusion}
% ============================================================

We have empirically confirmed the theoretical predictions of Bousquet and
Elisseeff~\cite{bousquet2002}:
\begin{enumerate}
    \item Increasing regularization strength $\lambda$ consistently improves algorithmic
    stability (lower $\betahat$) for both Ridge and Lasso.
    \item More stable models tend to generalize better, up to the point where
    over-regularization introduces excessive bias.
    \item Training set size and regularization have complementary effects on stability:
    both act to reduce $\betahat$, consistent with the bound $\beta = \mathcal{O}(1/\lambda m)$.
    \item L2 regularization (Ridge) is marginally more stable than L1 (Lasso) at the
    same $\lambda$, attributable to the smoothness of the quadratic penalty.
\end{enumerate}

These findings provide a mechanistic, stability-based interpretation of why
regularization helps generalization — an explanation that is complementary to, and in some
cases more informative than, classical VC-dimension arguments.
